# X Downloader 本地数据与隐私说明

X Downloader 采用本地优先设计，不提供云端服务，也不包含遥测或分析 SDK。

## 浏览器扩展处理的数据

- X/Twitter 当前页面 URL、帖子 ID、作者、发布时间和媒体缩略图 URL。
- 浏览器已经请求的 `video.twimg.com` HLS master URL。
- Helper 地址、配对令牌、显示模式和通知偏好。
- 设置页展示的本地下载目录。
- 浏览器提供的 User-Agent 字符串，用于保持媒体请求兼容。
- 当前标签页内的候选与任务状态。

扩展使用 `storage` 保存设置，使用 `notifications` 显示下载结果；只在 X/Twitter、`video.twimg.com` 和 `127.0.0.1` 范围内工作。

## Helper 保存的数据

- 高熵配对令牌。
- 所有已识别媒体候选、帖子上下文和媒体 playlist URL，直到用户删除历史。
- 所有下载任务、尝试记录、进度结果、失败分类、本地输出路径和文件大小。
- 设置页选择并由 Helper 保存的当前下载目录。
- 文件命名模板、并发数、失败重试次数和媒体候选对应的 User-Agent。
- 用户在网页管理台创建的标签、标签颜色和历史备注。
- 下载中的临时文件与完成的 MP4。

这些结构化数据默认位于用户配置目录的 `x-downloader/state/x-downloader.sqlite3`，数据库文件权限为 0600，所在状态目录权限为 0700。下载文件默认位于 `~/Downloads/X-Media/`。统计信息由本机数据库按需聚合，不发送遥测。

Helper 运行的网页管理台只从同一回环地址加载脚本和样式；数据接口需要配对令牌。网页把令牌放在当前标签页的 `sessionStorage`，不写入 Helper 数据库、URL或长期浏览器存储。

## 不会处理或上传的数据

- 不读取或保存浏览器 Cookie。
- 不读取 X 的账号密码、Cookie 或 Authorization 请求头。
- 不绕过 DRM、付费或私密访问控制。
- 不把视频、URL、令牌或使用记录上传到项目作者或第三方服务。

Helper 只监听 `127.0.0.1`，除公开健康检查外的 API 都要求 bearer token；媒体抓取仅允许 `https://video.twimg.com/*.m3u8`。

## 清理数据

网页管理台可以删除单条帖子历史及其数据库任务记录，或单独删除标签。退出 Helper 后删除 `stateDir` 可以清除候选、任务、设置、备注和标签；删除 `tokenFile` 会在下次启动时生成新令牌。删除浏览器扩展会清理其扩展存储。已经下载的 MP4 需要用户自行删除。

从 JSON 状态版本首次升级后，旧文件会以 `.migrated-v1.bak` 保留，便于核对和回退；它们包含与迁移前相同的本地数据，需要用户确认迁移无误后自行删除。
